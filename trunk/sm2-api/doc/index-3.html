3.2 Services and Modules

Following the principle of the Bridge design pattern, each service that Zookeeper provides will be implemented by one or more modules.

A service is a front-end API that the web application will use to interface with Zookeeper.

For example, a web mail application might use a "session" service to store users' session data.

A module is backend-end code that will perform the functions of a service. Abstracting the code that is actually performing the functions of a service allows for greater flexibility.

For example, the a "session" service could use a "MySQL" module, a "PostgreSQL" module, or a "flat files" module.
