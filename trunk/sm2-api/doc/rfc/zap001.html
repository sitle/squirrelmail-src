Tonight I made CVS changes to do the following:

  * Renamed lib directory to servies
  * Renamed examples directory to regtest (standing for
    regression tests - will explain more later)
  * Renamed all messages directories to message
    (though I'd rather do it now then later)
  * Added doc directory

Also, I wanted to pass out responsiblities a little as I have seen
things developing.

 Jon Tai: documentation manager
 Phillipe Mingo: dev of auth, i18n, and session services
 Wouter Teepe: dev of message service
 Paul Thompson: dev of logging and error handling services

Also, I will be overseeing and helping the rest of you in your
corresponding areas.

I am giving these "assignments" just to organize the work a little
more effectively. They by no means that we can't work on and help
each other in things.

Lastly, I just added a directory called doc/rfc to the repository.
Please write up your proposals for each of your work areas and put
them in here. They should be numbered, starting with 001.html, and
progressing from there. So that we don't have numbering conflicts, do
a 'touch xxx.html', 'cvs add xxx.html', and 'cvs commit xxx.html'
real quickly when you start on a proposal.

Now, I know that this might come across as too complicated and
overkill. However, all I intend this to be is that instead of writing
these big long messages to the list with our proposals, put them in
RFCs (remember, RFC = Request For Comments) and then send a message
to the list asking us to review it.

And of course, after comments are given, the RFCs will get updated,
etc. Later, the text from the RFCs can maybe be transferred into
perminent documentation when needed.

Thanks again for your help.
